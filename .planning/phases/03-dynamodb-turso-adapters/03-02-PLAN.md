**Step 2: Create SQLite-compatible schema** at `internal/repository/turso/migrations/001_create_tables.sql`:
```sql
CREATE TABLE IF NOT EXISTS conversations (
    id         TEXT PRIMARY KEY,
    partner_id TEXT NOT NULL,
    user_id    TEXT NOT NULL,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_conversations_list
    ON conversations (partner_id, user_id, updated_at DESC);

CREATE TABLE IF NOT EXISTS messages (
    id              TEXT PRIMARY KEY,
    conversation_id TEXT NOT NULL REFERENCES conversations(id),
    role            TEXT NOT NULL,
    content         TEXT NOT NULL,
    token_count     INTEGER NOT NULL,
    created_at      TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_messages_window
    ON messages (conversation_id, created_at DESC);
```
Note: SQLite uses TEXT for UUIDs and timestamps (stored as RFC3339 strings). INTEGER instead of INT. No TIMESTAMPTZ.

**Constructor:** `func New(ctx context.Context, url string, authToken string) (*TursoRepository, error)`
- Build connection string: `url + "?authToken=" + authToken`
- Open with `sql.Open("libsql", connStr)`
- Ping with `db.PingContext(ctx)` to verify connectivity
- Apply schema: split `schema` string by `;` and execute each statement (SQLite doesn't support multi-statement exec in all drivers) — OR use `db.ExecContext(ctx, schema)` and if that fails, split and exec individually
- Return `&TursoRepository{db: db}`

**Close:** `func (r *TursoRepository) Close() error { return r.db.Close() }`

**CreateConversation (mirrors Postgres):**

- Return `domain.Conversation{...}` with parsed values

**AppendMessage (mirrors Postgres — per D-08, raw sql.Query):**
- `id := uuid.New()`, `now := time.Now().UTC()`, `tc := len(content) / 4`
- `INSERT INTO messages (id, conversation_id, role, content, token_count, created_at) VALUES (?, ?, ?, ?, ?, ?)`
- Pass `id.String(), conversationID.String(), string(role), content, tc, now.Format(time.RFC3339Nano)`
- `UPDATE conversations SET updated_at = ? WHERE id = ?` with `now.Format(time.RFC3339Nano), conversationID.String()`
- Return `domain.Message{...}`

**LoadWindow (mirrors Postgres DESC+reverse):**

**ListConversations (mirrors Postgres):**

- Pass `partnerID.String(), userID.String()`
- Scan into string vars, parse UUIDs and timestamps
- Return `[]domain.Conversation{}` (empty slice not nil)

```go
func setupTurso(t *testing.T) *TursoRepository {
    t.Helper()
    url := os.Getenv("TURSO_URL")
    token := os.Getenv("TURSO_AUTH_TOKEN")
    if url == "" || token == "" {
        t.Skip("Skipping Turso tests: TURSO_URL and TURSO_AUTH_TOKEN not set")
    }
    ctx := context.Background()
    repo, err := New(ctx, url, token)
    require.NoError(t, err)
    t.Cleanup(func() { repo.Close() })
    return repo
}
```

**Important: table cleanup between test runs.** Since Turso uses a real cloud database, tests must clean up after themselves. Add a cleanup helper that deletes test data after each test (DELETE FROM messages WHERE conversation_id IN (...); DELETE FROM conversations WHERE id IN (...)), OR use unique partnerID/userID per test so ListConversations doesn't see stale data.

**Approach: Use unique UUIDs per test** (uuid.New() for partnerID and userID in each test). This avoids cross-test interference without needing explicit cleanup, since ListConversations is scoped by partnerID+userID.
