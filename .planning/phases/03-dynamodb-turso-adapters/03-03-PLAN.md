type BackendMeta struct {
    Name      string // e.g., "postgres", "dynamodb", "turso"
    Transport string // e.g., "pgx/v5 (local container)", "aws-sdk-go-v2 (LocalStack)", "libsql:// (remote, internet)"
    Note      string // optional, e.g., "Latency includes internet round-trip to Turso Cloud"
}
```

Update `PrintResults` signature to accept `BackendMeta` instead of bare `backend string`:

func PrintResults(meta BackendMeta, profile string, iterations int, seed int64, results []ScenarioResult)

```

For postgres/dynamodb, the Note field is empty (no extra note printed).

**Step 2: Update `main.go` — multi-backend dispatch:**

**Backend validation (per D-13):** Replace the single-string validation with a multi-backend parser:
- Parse `--backend` value: if "all", expand to `["postgres", "dynamodb", "turso"]`
- If comma-separated (e.g., "postgres,dynamodb"), split into list
- If single value, wrap in list
- Valid values: "postgres", "dynamodb", "turso", "all"
- Invalid value: print error and exit

**Backend execution loop:** Iterate over the list of backends. For each backend:

**Postgres backend (existing logic, refactored into helper):**
- Start testcontainer, create postgres.New, run benchmarks, print results
- BackendMeta: `{Name: "postgres", Transport: "pgx/v5 (local container)", Note: ""}`

**DynamoDB backend (per D-10):**
- Start LocalStack container: `localstack.Run(ctx, "localstack/localstack:latest")`
- Get endpoint: `host, _ := ctr.Host(ctx)` + `port, _ := ctr.MappedPort(ctx, "4566/tcp")` -> `http://host:port`
- Create repo: `dynamodb.New(ctx, endpoint)`
- Run benchmarks with same runner pattern
- BackendMeta: `{Name: "dynamodb", Transport: "aws-sdk-go-v2 (LocalStack)", Note: ""}`
- Cleanup: `testcontainers.TerminateContainer(ctr)` + `repo.Close()`

**Turso backend (per D-11, D-14):**
- Read env vars: `os.Getenv("TURSO_URL")`, `os.Getenv("TURSO_AUTH_TOKEN")`
- If either is empty AND we're in `--backend all` mode: print warning `"Skipping turso: TURSO_URL and TURSO_AUTH_TOKEN not set"` and continue to next backend (per D-14)
- If either is empty AND turso was explicitly requested (not via "all"): print error and exit
- Create repo: `turso.New(ctx, url, token)`
- Run benchmarks
- BackendMeta: `{Name: "turso", Transport: "libsql:// (remote, internet)", Note: "Latency includes internet round-trip to Turso Cloud"}` (per D-15)
- Cleanup: `repo.Close()`

**Side-by-side output (per D-16):** Each backend prints its own results section with header. When multiple backends run, there's a blank line between sections. Example output:
```
Backend: postgres | Transport: pgx/v5 (local container) | Profile: medium | Iterations: 100 | Seed: 42

SCENARIO   P50   P95   P99   COUNT

Backend: dynamodb | Transport: aws-sdk-go-v2 (LocalStack) | Profile: medium | Iterations: 100 | Seed: 42

SCENARIO   P50   P95   P99   COUNT
```

**Dry-run per backend (extend existing dry-run):**
- Postgres: existing behavior (container + connectivity + schema + seed + sample query)
- DynamoDB: start LocalStack container, create table, insert test conversation, query it
- Turso: connect to cloud, apply schema, insert test conversation, query it (or skip if env vars missing in `--backend all` mode)
