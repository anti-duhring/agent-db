    - `ComputeProjections(scale ScaleConfig, cost CostConfig) []BackendCostProjection` function that returns 3 projections:

      **DynamoDB calculation:**
      - dailyWrites = scale.Users * scale.ConvosPerUser * scale.MsgsPerDay
      - monthlyWRU = dailyWrites * cost.DynamoDBWRUPerAppend * 30
      - monthlyWRUCost = float64(monthlyWRU) / 1_000_000 * cost.DynamoDBWRUPerMillion
      - dailyReads = dailyWrites * 2 (LoadWindow + ListConversations estimate)
      - monthlyRRU = dailyReads * 30
      - monthlyRRUCost = float64(monthlyRRU) / 1_000_000 * cost.DynamoDBRRUPerMillion
      - estimatedStorageGB = float64(dailyWrites*30*12) * 0.002 / 1024 (rough: 2KB avg msg, 12 months accumulation)
      - storageCost = estimatedStorageGB * cost.DynamoDBStoragePerGB
      - IO cost = monthlyWRUCost + monthlyRRUCost
      - Total = IO + storageCost (no instance compute for on-demand)

      **RDS calculation:**
      - instanceCost = float64(cost.RDSHoursPerMonth) * cost.RDSInstanceHourly
      - storageCost = same estimatedStorageGB formula * cost.RDSStoragePerGB
      - Total = instanceCost + storageCost

      **Turso calculation:**
      - monthlyRowWrites = dailyWrites * 30 (1 write = ~3 row writes for INSERT + UPDATE)
      - monthlyRowReads = dailyReads * 30 * 20 (LoadWindow 20 rows scanned per read, ListConversations 2x multiplier per Pitfall 5)
      - Determine tier: if within starter limits, use starter; else use scaler; if exceeds scaler, compute overage
      - Total = plan cost + any overage
