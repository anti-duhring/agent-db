    Create `internal/generator/generator.go` with package `generator`:

    1. Define profile constants (per D-11):
       ```go
       type Profile struct {
           Name          string
           Conversations int
           Messages      int // per conversation
       }

       var (
           Small  = Profile{Name: "small",  Conversations: 5,  Messages: 10}
           Medium = Profile{Name: "medium", Conversations: 10, Messages: 500}
           Large  = Profile{Name: "large",  Conversations: 10, Messages: 5000}
       )
       ```

    2. Define `Generator` struct holding `faker *gofakeit.Faker` (NEVER use global gofakeit functions — per Pitfall 1).

    3. Constructor `func New(seed int64) *Generator` using `gofakeit.New(seed)` (per D-09).

    4. Define result type to hold generated data:
       ```go
       type GeneratedData struct {
           Conversations []domain.Conversation
           Messages      map[uuid.UUID][]domain.Message // keyed by conversation ID
       }
       ```

    5. Main generation method `func (g *Generator) Generate(partnerID, userID uuid.UUID, profile Profile) GeneratedData`:
       - For each conversation (0 to profile.Conversations-1):
         - Generate conversation UUID using the faker's seeded random source. Use `uuid.Must(uuid.NewRandomFromReader(g.faker))` if gofakeit implements `io.Reader`, otherwise use a deterministic approach: generate 16 random bytes via `g.faker.Uint8()` calls and construct a UUID. The key requirement is that the UUID must be deterministic from the seed.
         - IMPORTANT: gofakeit's `*Faker` struct has a `Rand` field of type `*rand.Rand`. Use `uuid.Must(uuid.NewRandomFromReader(g.faker.Rand))` — rand.Rand implements io.Reader in Go. If that doesn't work, fall back to generating 16 bytes via loop of `g.faker.Uint8()` and calling `uuid.FromBytes(bytes)`.
         - Set `now` using a deterministic base time, e.g., `time.Date(2026, 1, 1, 0, 0, 0, 0, time.UTC)` plus a faker-generated offset. This ensures deterministic timestamps.
         - Build `domain.Conversation{ID: convID, PartnerID: partnerID, UserID: userID, CreatedAt: baseTime, UpdatedAt: baseTime}`

       - For each message in the conversation (0 to profile.Messages-1):
         - Generate message UUID deterministically (same approach as conversation UUID)
         - Set role via alternation (per D-07, Research Role Alternation pattern):
           `if i%2 == 0 { role = domain.RoleUser } else { role = domain.RoleAssistant }`
         - Generate content via `g.generateContent()` (see below)
         - Compute `tokenCount := len(content) / 4` (per D-12)
         - Set `CreatedAt` as base time + message index offset (e.g., add `i * time.Minute`) for chronological ordering
         - Update conversation's `UpdatedAt` to the last message's `CreatedAt`

    6. Content generation method `func (g *Generator) generateContent() string` (per D-10, DATA-03):
       - Use `g.faker.Paragraph(1, 3, 10, " ")` or similar to generate base content. Note: gofakeit v7 `Paragraph` signature is `Paragraph(paragraphCount, sentenceCount, wordCount int, separator string) string`.
       - Pad if under 100 chars: `for len(content) < 100 { content += " " + g.faker.Sentence(10) }`
       - Truncate at 2000 chars: `if len(content) > 2000 { content = content[:2000] }`
       - Return content

    7. Helper `func tokenCount(content string) int` returning `len(content) / 4`.

    Imports: `"time"`, `"github.com/brianvoe/gofakeit/v7"`, `"github.com/anti-duhring/agent-db/internal/domain"`, `"github.com/google/uuid"`.

    CRITICAL: NEVER call `gofakeit.Sentence(...)` or `gofakeit.Paragraph(...)` as package-level functions. ALWAYS call `g.faker.Sentence(...)` and `g.faker.Paragraph(...)` on the seeded instance.

    Then create `internal/generator/generator_test.go` with tests covering all behaviors above. For the determinism test, run Generate twice with the same seed and compare all fields. For content length, iterate all messages in a generated Small profile and assert 100 <= len(content) <= 2000.
