  <action>
    Create `internal/repository/memory/memory.go` with package `memory`:

    1. Define `MemoryRepository` struct with:
       - `mu sync.RWMutex`
       - `conversations map[uuid.UUID]domain.Conversation` (keyed by conversation ID)
       - `messages map[uuid.UUID][]domain.Message` (keyed by conversation ID, slice of messages in append order)

    2. Constructor `func New() *MemoryRepository` initializing both maps with `make()`.

    3. Compile-time interface check (per Pattern 5):
       `var _ repository.ChatRepository = (*MemoryRepository)(nil)`

    4. `CreateConversation`:
       - Write-lock (`mu.Lock()` / `defer mu.Unlock()`)
       - Generate `id := uuid.New()`
       - Set `now := time.Now()`
       - Build `domain.Conversation{ID: id, PartnerID: partnerID, UserID: userID, CreatedAt: now, UpdatedAt: now}`
       - Store in `r.conversations[id]`
       - Initialize empty message slice: `r.messages[id] = []domain.Message{}`
       - Return the conversation and nil error

    5. `AppendMessage`:
       - Write-lock
       - Check `r.conversations[conversationID]` exists; if not, return error `fmt.Errorf("conversation %s not found", conversationID)`
       - Generate `id := uuid.New()`, `now := time.Now()`
       - Compute `tokenCount := len(content) / 4` (per D-12)
       - Build `domain.Message{ID: id, ConversationID: conversationID, Role: role, Content: content, TokenCount: tokenCount, CreatedAt: now}`
       - Append to `r.messages[conversationID]`
       - Update conversation's `UpdatedAt` to `now` (must re-assign into the map since structs are values)
       - Return message and nil error

    6. `LoadWindow`:
       - Read-lock (`mu.RLock()` / `defer mu.RUnlock()`)
       - Check conversation exists; if not, return error
       - Get messages slice for the conversation
       - If `n >= len(msgs)`, return a copy of the full slice
       - Otherwise return a copy of `msgs[len(msgs)-n:]` (last N messages, chronological order)
       - IMPORTANT: Return a copy (not a reference to the internal slice) to prevent callers from mutating internal state

    7. `ListConversations`:
       - Read-lock
       - Iterate `r.conversations`, collect those where `c.PartnerID == partnerID && c.UserID == userID`
       - Sort result by `UpdatedAt` descending: `sort.Slice(result, func(i, j int) bool { return result[i].UpdatedAt.After(result[j].UpdatedAt) })`
       - Return result slice (empty `[]domain.Conversation{}` if none found, not nil)
       - Return nil error

    Imports: `"context"`, `"fmt"`, `"sort"`, `"sync"`, `"time"`, `"github.com/anti-duhring/agent-db/internal/domain"`, `"github.com/anti-duhring/agent-db/internal/repository"`, `"github.com/google/uuid"`.

    Note: The `context.Context` parameter is accepted but not used in the in-memory implementation (no cancellation or timeouts needed). This is fine — the interface requires it for database adapters.

    Then create `internal/repository/memory/memory_test.go` with tests covering all behaviors listed above. Use `testing` stdlib. Run with `-race` flag to verify concurrent safety.
