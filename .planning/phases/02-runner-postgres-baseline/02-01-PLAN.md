    2. Create `internal/repository/postgres/migrations/001_create_tables.sql` per D-01 and D-02:
       ```sql
       CREATE TABLE IF NOT EXISTS conversations (
           id         UUID PRIMARY KEY,
           partner_id UUID NOT NULL,
           user_id    UUID NOT NULL,
           created_at TIMESTAMPTZ NOT NULL,
           updated_at TIMESTAMPTZ NOT NULL
       );

       CREATE INDEX IF NOT EXISTS idx_conversations_list
           ON conversations (partner_id, user_id, updated_at DESC);

       CREATE TABLE IF NOT EXISTS messages (
           id              UUID PRIMARY KEY,
           conversation_id UUID NOT NULL REFERENCES conversations(id),
           role            TEXT NOT NULL,
           content         TEXT NOT NULL,
           token_count     INT  NOT NULL,
           created_at      TIMESTAMPTZ NOT NULL
       );

       CREATE INDEX IF NOT EXISTS idx_messages_window
           ON messages (conversation_id, created_at DESC);
       ```

    3. Create `internal/repository/postgres/postgres.go`:
       - Package `postgres`
       - Embed schema via `//go:embed migrations/001_create_tables.sql` into `var schema string` (per D-03)
       - Compile-time check: `var _ repository.ChatRepository = (*PostgresRepository)(nil)`
       - `PostgresRepository` struct with `pool *pgxpool.Pool` field
       - `New(ctx context.Context, connString string) (*PostgresRepository, error)`:
         - Parse config via `pgxpool.ParseConfig(connString)`
         - Set `config.AfterConnect` callback that prepares 5 statements (per D-04):
           - `"insert_conversation"`: `INSERT INTO conversations (id, partner_id, user_id, created_at, updated_at) VALUES ($1, $2, $3, $4, $5)`
           - `"insert_message"`: `INSERT INTO messages (id, conversation_id, role, content, token_count, created_at) VALUES ($1, $2, $3, $4, $5, $6)`
           - `"load_window"`: `SELECT id, conversation_id, role, content, token_count, created_at FROM messages WHERE conversation_id = $1 ORDER BY created_at DESC LIMIT $2`
           - `"list_conversations"`: `SELECT id, partner_id, user_id, created_at, updated_at FROM conversations WHERE partner_id = $1 AND user_id = $2 ORDER BY updated_at DESC`
           - `"update_conversation_updated_at"`: `UPDATE conversations SET updated_at = $1 WHERE id = $2`
         - Create pool via `pgxpool.NewWithConfig(ctx, config)`
         - Apply schema by executing the embedded SQL string via `pool.Exec(ctx, schema)`
         - Return `&PostgresRepository{pool: pool}, nil`
       - `Close()`: calls `r.pool.Close()`
       - `CreateConversation(ctx, partnerID, userID)`:
         - Generate `id := uuid.New()`, `now := time.Now().UTC()`
         - Execute prepared statement `"insert_conversation"` with `(id, partnerID, userID, now, now)`
         - Return `domain.Conversation{ID: id, PartnerID: partnerID, UserID: userID, CreatedAt: now, UpdatedAt: now}, nil`
       - `AppendMessage(ctx, conversationID, role, content)`:
         - Generate `id := uuid.New()`, `now := time.Now().UTC()`, `tc := len(content) / 4`
         - Execute prepared statement `"insert_message"` with `(id, conversationID, string(role), content, tc, now)`
         - Execute prepared statement `"update_conversation_updated_at"` with `(now, conversationID)`
         - Return `domain.Message{ID: id, ConversationID: conversationID, Role: role, Content: content, TokenCount: tc, CreatedAt: now}, nil`
       - `LoadWindow(ctx, conversationID, n)` per D-05:
         - Query using prepared statement `"load_window"` with `(conversationID, n)`
         - Scan rows into `[]domain.Message` using `pgx.CollectRows` with `pgx.RowToStructByPos[domain.Message]`
           - NOTE: If RowToStructByPos does not work due to field ordering/types, use manual row scanning instead: `rows.Scan(&msg.ID, &msg.ConversationID, &msg.Role, &msg.Content, &msg.TokenCount, &msg.CreatedAt)`. The Role field needs scanning into a string then casting to `domain.Role`.
         - Reverse the slice in-place (oldest-first output)
         - Return the reversed slice
       - `ListConversations(ctx, partnerID, userID)`:
         - Query using prepared statement `"list_conversations"` with `(partnerID, userID)`
         - Scan rows into `[]domain.Conversation`
         - If no rows, return `[]domain.Conversation{}, nil` (empty slice, not nil)
